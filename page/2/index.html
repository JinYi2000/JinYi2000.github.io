同源策略

如果协议、域名或者端口有一个不同就是跨域，Ajax 请求会失败。

那么是出于什么安全考虑才会引入这种机制呢？ 其实主要是用来防止 CSRF 攻击的。简单点说，CSRF 攻击是利用用户的登录态发起恶意请求。

也就是说，没有同源策略的情况下，A 网站可以被任意其他来源的 Ajax 访问到内容。如果你当前 A 网站还存在登录态，那么对方就可以通过 Ajax 获得你的任何信息。当然跨域并不能完全阻止 CSRF。

跨域

场景：前端服务在一个服务器上 后端在另一个服务器上

cors

在接口的服务端修改响应头

服务端设置 Access-Control-Allow-Origin

该属性表明哪些域名可以访问资源

JSONP

href/src 请求下来的js脚本、css文件都不存在跨域问题

只有ajax请求来的数据才存在跨域问题

通过请求js脚本来获取数据。 在请求的过程中传入一个回调函数。 脚本自动执行回调函数及其中的参数。

postMessage

document.domain

vue 跨域的原理

浏览器是禁止跨域的，但是服务端不禁止，在本地运行npm run dev等命令时实际上是用node运行了一个服务器，因此proxyTable实际上是将请求发给自己的服务器，再由服务器转发给后台服务器，做了亦曾代理，因为不会出现跨域问题。

执行栈

存储函数调用的栈结构

JS是单线程的

多线程处理dom可能会出现问题（一个线程增加节点，另一个线程删除节点）

浏览器中的eventloop

分出主次之分

  1. 执行环境会被顺序加入执行栈中

  2. 遇到异步代码的时候，挂起，加入task中 。一旦执行栈为空，eventloop就会从task队列中拿出代码放入执行栈执行。

  3. task队列分为两种。微任务microtask和宏任务macrotask

    微任务： process.nextTick promise

    宏任务；script setTimeout setInterval I/O UI rendering

    微任务优先级最高

执行栈 > 微任务 > 宏任务

如果微任务队列中有微任务，则一次执行完所有微任务。不然就去执行宏任务。执行完一个宏任务之后再查看微任务队列。

定时器的执行：定时结束之后加入到宏任务队列中

Promise 同步 + 异步（微任务）
