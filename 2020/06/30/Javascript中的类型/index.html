Javascript中的类型

undefined

表示未定义 只有一个值undefined 任何变量在赋值前是undefined类型，值为undefined

但是undefined是一个变量而不是关键字 有时候用void 0来获取undefined的值

null

null 的意思就是空 是JS关键字 不用担心不小心被篡改

boolean

表示逻辑上的真假 true false

string

String 的意义并非“字符串”，而是字符串的 UTF16 编码

number

具有精度限制

因此浮点数运算具有精度问题 相差 了一个微小的值 ：Number.EPSILON

symbol

一个永远不会重复的标记

用法：

  • var hd = Symbol(‘mzh’)

  • var hd = Symbol.for(‘jinyi’) 这样声明就可以重复了

用途

eg:防止字符串耦合

object

对象，属性的集合

除开原始类型之外都是对象

储存的是指针

原始类型储存的是值

属性

  • 数据属性
  • 访问器属性
  • key可以是字符串或Symbol类型

Number String Boolean

  • 和new搭配时产生对象
  • 直接调用时强制类型转换

类型转换

一般用 === 比较
