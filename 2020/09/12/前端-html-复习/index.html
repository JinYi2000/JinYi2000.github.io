1、前端SEO?

  • 前端seo 分为title description keywords. 而且这三个权重逐渐缩小。

  • 要语义化的html代码，而且重要的html代码要放在前面。

  • 重要的内容不要用js输出，因为js输出的内容不会被爬虫抓到。

  • 少用iframe，因为iframe中的东西不会被搜索引擎抓取。

  • 非修饰性图片加alt

    alt用于在图片无法显示的时候替代图片

  • 提高网站的速度，因为网站的速度会影响搜索引擎的排序

2、img的title和alt的区别？

title是鼠标滑过图片时候的文字提示。而alt是图片不显示时候的替代文字。

3、http的几种请求方法及其用途？

  • get:用于请求服务器上的某一资源
  • post:向url指定的资源提交数据或附加新的数据
  • put:也是向服务器提交数据。 与post不同的地方在于指定了资源在服务器上的位置。
  • Delete:删除了服务器上的资源
  • optons:用于获取当前url所支持的方法。
  • Trace:
  • Connect:
  • Head:只返回请求的首部

http的工作原理

http请求/响应的步骤

  1. 客户端连接到web服务器

    与服务器端口建立套接字连接

  2. 发送http请求

    通过tcp套接字，客户端向服务器发送文本的请求报文。

    一个请求报文由请求行、请求头部、空行的请求数据四部分组成

  3. http接受请求并返回http响应

    一个响应由状态行、响应头部、空行和响应数据四部分组成

  4. 释放连接tcp连接

    若connection 模式为close，则服务器主动关闭TCP连接，客户端被动关闭连接，释放TCP连接;若connection 模式为keepalive，则该连接会保持一段时间，在该时间内可以继续接收请求;

  5. 客户端浏览器解析html内容

http响应消息response

状态行 消息报头 空行和响应正文

请求返回的状态码

200 OK ：响应正文中返回数据

204 No Content :操作成功但不在响应正文中返回数据

304 Not Modified:重定向 测试实体自上次以来是否有被修改

403 Forbidden : 客户端错误

401 Unauthorized ：客户端错误

413 Payload Too Large ：（客户端错误） 当请求长度过长时，出现此结果。

400 BadRequest ：（客户端错误） 当参数无效时，出现此结果。

404 Not Found（客户端错误） 当资源不存在时，出现此结果。

405 Method Not Allowed（客户端错误）由于方法和资源组合不正确而出现此错误。 例如，您不能对一个实体集合使用 DELETE 或 PATCH。

412 Precondition Failed 客户端错误

501 Not Implemented（服务器错误） 当未实施某个请求的操作时，出现此结果。

503 Service Unavailable（服务器错误） 当 Web API 服务不可用时，出现此结果。

get与post

get方法请求的数据直接填充在请求报文的url上（可以用来传输一些可以公开的参数信息，解析也比较方便）

post方法提交的数据会附在正文上（POST方法可以用来提交一个用户的敏感信息）
